# Security Policy

## Supported Versions
- Only the latest major version is supported (see CHANGELOG.md).

## Reporting a Vulnerability
If you discover a security vulnerability in ClearScan, please report it privately to the maintainers at security@deusolutions.ru or via GitHub Issues (with the `security` label).

**Do not disclose vulnerabilities publicly until a fix is released.**

## Best Practices
- Never commit real tokens, passwords, or secrets.
- Use environment variables or CI/CD secrets for sensitive data.
- Follow [docs/security.md](docs/security.md) for hardening recommendations.
